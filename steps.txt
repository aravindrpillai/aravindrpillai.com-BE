Update OS and Install necessary libs
    apt update && apt upgrade -y
    apt install -y python3 python3-venv python3-pip git nginx ufw

Get the code
    git clone https://github.com/aravindrpillai/aravindrpillai.com-BE.git .

Create virtual environment
    python3 -m venv /root/venv

Activate venv
    source /root/venv/bin/activate

Verify:
    which python
    #OP should be:  /root/venv/bin/python

Install dependencies
    pip install --upgrade pip
    pip install -r requirements.txt
    pip install django gunicorn
    pip freeze > requirements.txt

Edit settings.py
    nano <project>/settings.py
    #and configure the allowed host and cors and DEBUG = False

Run migrations
    python manage.py migrate

Test locally (DO NOT SKIP)
    python manage.py runserver 0.0.0.0:8000
    curl http://127.0.0.1:8000/
And if it works, stop server which we ran using above command:
    CTRL + C

Configure GUNICORN (APP SERVER)
Open django service    
    nano /etc/systemd/system/django.service

Paste below:
-------------DONT TAKE THIS LINE-----------------------------------------
[Unit]
Description=Django Application (Gunicorn)
After=network.target

[Service]
User=root
Group=www-data
WorkingDirectory=/root/myproject
ExecStart=/root/venv/bin/gunicorn \
    --workers 3 \
    --bind 127.0.0.1:8000 \
    <project>.wsgi:application

Restart=always

[Install]
WantedBy=multi-user.target
-------------DONT TAKE THIS LINE-----------------------------------------



Start Gunicorn
    systemctl daemon-reload
    systemctl enable django
    systemctl restart django
    systemctl status django


After opening the port in AWS of Vultr:
1. check if 80 is open
        ufw status
2. If not do the below    
    ufw allow 80/tcp
    ufw reload
3. then check the status again
    ufw status
